# Пульт менеджера (operator console)

Отдельный веб-экран для менеджеров: живые диалоги Киры в стиле мессенджера.
Менеджер может **вмешаться в чат** (Кира замолкает), **писать клиенту** в Telegram
прямо отсюда и **вернуть управление Кире**. Всё пишется в БД и в amoCRM.

Фронтенд статический (HTML/CSS/JS) — как веб-версия Киры. Работает поверх того же
бэкенда (FastAPI), где живёт бот и админка.

## Как работает

- Менеджер отправляет сообщение из пульта → чат переходит в **ручной режим**
  (`chat_control = manager`), Кира перестаёт отвечать автоматически.
- Клиенту один раз уходит строка «К диалогу подключился менеджер» (без имени).
- Ответы клиента приходят в тот же диалог, Кира молчит.
- Кнопка **«Вернуть Кире»** снимает ручной режим. Есть и авто-возврат: если
  менеджер молчит дольше `MANAGER_AUTO_RETURN_HOURS` (по умолчанию 3 ч), Кира
  сама подхватывает диалог.

## Настройка

1. В `config.js` укажите адрес бэкенда (`BACKEND_URL`) — домен задеплоенного
   FastAPI (тот же, где `/admin`). Локально — `http://localhost:8000`.
2. На бэкенде задайте переменные окружения:
   - `ADMIN_SECRET` — ключ доступа (его вводят менеджеры при входе в пульт).
   - `CORS_ORIGINS` — домен, где размещён пульт, напр.
     `https://operator.example.com` (можно несколько через запятую).
   - `MANAGER_AUTO_RETURN_HOURS` — часы до авто-возврата Кире (по желанию, деф. 3).
3. Разместите папку `manager_console/` на любом статическом хостинге со своим
   доменом (GitHub Pages / Netlify / VPS). Обязательно по HTTPS.

## Вход

Менеджер открывает сайт → вводит **ключ доступа** (`ADMIN_SECRET`). Адрес сервера
можно переопределить в блоке «Адрес сервера», если фронт и бэк на разных доменах.

## Безопасность

Это живые переписки клиентов (ПДн). Держите пульт **только по HTTPS**, ограничьте
`CORS_ORIGINS` доменом пульта и не публикуйте ключ доступа.
